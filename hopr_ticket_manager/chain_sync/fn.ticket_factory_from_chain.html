Expand description

Creates a HoprTicketFactory backed by a temporary RedbStore and pre-seeds it from the node’s current outgoing channels.

Use this when building an edge (entry/exit) node that issues tickets but does not relay them. For relay nodes that also manage incoming tickets, use ticket_manager_from_chain instead.
